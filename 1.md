**Game state machine:** `waiting` вҶ’ `playing` вҶ’ `round_result` вҶ’ `playing` вҶ’ ... вҶ’ `finished`

### `game.js` (246 lines) вҖ” GameRoom class

**Constructor:** `GameRoom(code, genres)` вҖ” in-memory room with settings, players, tracks.

**Scoring system:**
- Correct answer: **100 pts** base
- Time bonus: **10 pts per second** remaining (max 150)
- Max per round: 100 + (roundTime * 10) e.g. 15s вҶ’ 250 max

**Fuzzy answer matching (`isCorrectGuess`):**
- Normalize: lowercase, trim, strip punctuation, remove `(feat. ...)` / `(remastered)` / `(version)`
- Exact match вҶ’ correct
- One contains the other вҶ’ correct
- Levenshtein distance вүӨ 2 вҶ’ correct
- Distance вүӨ 25% of string length вҶ’ correct

**State machine:**
```
startGame() вҶ’ fetch Spotify tracks вҶ’ startRound()
startRound() вҶ’ search YouTube for track video + pre-fetch next вҶ’ set timer (+3s buffer) вҶ’ state=playing
submitAnswer() вҶ’ fuzzy match check вҶ’ calc points вҶ’ return result
endRound() вҶ’ show answer вҶ’ pause (customizable, 2-15s) вҶ’ next round or endGame()
```

**Field visibility in `getState()`:**
- `playing`: sends `youtubeVideoId`, `trackId`, `roundTime` (NOT the track name вҖ” players guess)
- `round_result`: sends `correctAnswer`, `artist`, `albumImage`, `pauseTimeLeft`
- `finished`: sends `rankings` array

**Settings:** rounds (3-25), roundTime (8-30s), pauseTime (2-15s, default 4s)

### `spotify.js` (102 lines) вҖ” Spotify API client

**Auth:** Client Credentials flow via `POST accounts.spotify.com/api/token`, cached with 2min buffer.

**`fetchWithRetry(url, retries=3)`:** Handles:
- 429 вҶ’ exponential backoff with `Retry-After`
- 401 вҶ’ force re-auth, retry
- All other errors вҶ’ throw with API message

**`getTracksByGenre(genre, count=10)`:** Paginates `GET /v1/search?q=genre:GENRE&type=track&limit=10` (max 100 results per genre). Returns `{id, name, artist, albumImage}`. No `preview_url` filter needed (audio from YouTube).

**20 genres:** pop, rock, hip-hop, r-n-b, electronic, jazz, classical, country, metal, indie, alternative, soul, funk, blues, reggae, punk, latin, dance, edm, acoustic

### `youtube.js` (61 lines) вҖ” YouTube Data API v3

**`searchYouTubeVideo(name, artist)`:**
- Cached (Map, max 500 entries)
- Queries `GET /v1/search?q={name}+{artist}+audio&type=video&maxResults=1`
- Returns `videoId` or `null`
- 429/403 вҶ’ returns null gracefully
- Requires `YOUTUBE_API_KEY`

### `auth.js` (96 lines) вҖ” Discord OAuth + JWT

**`handleDiscordCallback(code)`:**
1. Exchange code for token via Discord OAuth2
2. Fetch `GET /users/@me`
3. Upsert user in DB
4. Check `ADMIN_DISCORD_IDS` env var вҖ” if user's Discord ID in list вҶ’ role = `admin`
5. Sign JWT (7d expiry)

**Middleware:**
- `authenticate` вҖ” verifies `Authorization: Bearer <jwt>` вҶ’ `req.user = {userId, role}`
- `requireAdmin` вҖ” extends authenticate + checks `role === 'admin'`

### `db.js` (84 lines) вҖ” Database abstraction

**Dual backend:**
- `DATABASE_URL` set вҶ’ PostgreSQL via `pg` (Railway)
- No `DATABASE_URL` вҶ’ SQLite via `better-sqlite3` (local dev)

**Tables:**
```sql
users (id, discord_id UNIQUE, username, avatar_url, role DEFAULT 'user', created_at)
game_scores (id, user_id FK, game_code, score, total_rounds, played_at)
friendships (user_id FK, friend_id FK, status DEFAULT 'pending', created_at, PK(user_id, friend_id))
```

**Exports:** `generateId(), query(), get(sql, params), all(sql, params), run(sql, params)` вҖ” all async, works with both DB engines.

## Frontend вҖ” app/

### `page.tsx` (Home)

- Top 5 leaderboard fetched from API, linked to full leaderboard
- Bottom nav: **Account** / **Leaderboard**

### `game/[code]/page.tsx` вҖ” Core game UI

**Sub-components:**

| Component | State | What it shows |
|-----------|-------|---------------|
| `WaitingRoom` | `waiting` | Player list, genre chips, rounds/time/pause selectors (host only), Start button |
| `PlayingPhase` | `playing` | Local timer (starts when YouTube plays), 48-bar visualizer (motion spring), progress bar, guess input with auto-focus + Enter submit |
| `RoundResult` | `round_result` | Correct answer, artist, player scores, pause countdown timer |
| `GameFinished` | `finished` | Rankings podium, Play Again + Save Score buttons |
| `YouTubePlayer` | hidden | Hidden `<div>` with YouTube IFrame API, 0Г—0 size, auto-plays, fires `onPlaying` callback |
| `Visualizer` | any | 48 `motion.div` bars, spring animation, gradient colors, heights randomized every 120ms |
| `ProgressBar` | any | `motion.div` width based on `getCurrentTime()/getDuration()`, polled every 200ms |

**Timer sync:** Frontend starts local countdown only when YouTube fires `onPlay` event вҶ’ timer is synced with actual music start. Backend adds 3s buffer to round timeout.

**Auto-focus:** Input ref is focused when `onPlay` fires.

**Enter key:** `onKeyDown={e => e.key === 'Enter' && onSubmit()}`

**Polling:** `fetchGameState(code)` every 1s. On `playing` вҶ’ set `youtubeVideoId`, reset local timer. YouTube player polls `getCurrentTime()/getDuration()` every 200ms.

**Error handling:** 5 failed polls вҶ’ show error with server URL.

### `login/page.tsx` вҖ” Discord OAuth

### `leaderboard/page.tsx`

### `admin/page.tsx`

### `create/page.tsx` + `join/page.tsx`
