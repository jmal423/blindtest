**Game state machine:** `waiting` → `playing` → `round_result` → `playing` → ... → `finished`

### `game.js` (196 lines) — GameRoom class

**Constructor:** `GameRoom(code, genres)` — in-memory room with settings, players, tracks.

**Scoring system:**
- Correct answer: **100 pts** base
- Time bonus: **10 pts per second** remaining (max 150)
- Max per round: 100 + (roundTime * 10) e.g. 15s → 250 max

**State machine:**
```
startGame() → fetch Spotify tracks → startRound()
startRound() → search YouTube for track video + pre-fetch next → set timer → state=playing
submitAnswer() → check name match → calc points → return result
endRound() → show answer → pause 4s → next round or endGame()
```

**Field visibility in `getState()`:**
- `playing`: sends `youtubeVideoId`, `trackId` (NOT the track name — players guess)
- `round_result`: sends `correctAnswer`, `artist`, `albumImage`
- `finished`: sends `rankings` array

**Settings:** rounds (3-25), roundTime (8-30s), pauseTime (4s fixed)

### `spotify.js` (102 lines) — Spotify API client

**Auth:** Client Credentials flow via `POST accounts.spotify.com/api/token`, cached with 2min buffer.

**`fetchWithRetry(url, retries=3)`:** Handles:
- 429 → exponential backoff with `Retry-After`
- 401 → force re-auth, retry
- All other errors → throw with API message

**`getTracksByGenre(genre, count=10)`:** Paginates `GET /v1/search?q=genre:GENRE&type=track&limit=10` (max 100 results per genre). Returns `{id, name, artist, albumImage}`. No `preview_url` filter needed (audio from YouTube).

**20 genres:** pop, rock, hip-hop, r-n-b, electronic, jazz, classical, country, metal, indie, alternative, soul, funk, blues, reggae, punk, latin, dance, edm, acoustic

### `youtube.js` (61 lines) — YouTube Data API v3

**`searchYouTubeVideo(name, artist)`:**
- Cached (Map, max 500 entries)
- Queries `GET /v1/search?q={name}+{artist}+audio&type=video&maxResults=1`
- Returns `videoId` or `null`
- 429/403 → returns null gracefully
- Requires `YOUTUBE_API_KEY`

### `auth.js` (96 lines) — Discord OAuth + JWT

**`handleDiscordCallback(code)`:**
1. Exchange code for token via Discord OAuth2
2. Fetch `GET /users/@me`
3. Upsert user in DB
4. Check `ADMIN_DISCORD_IDS` env var — if user's Discord ID in list → role = `admin`
5. Sign JWT (7d expiry)

**Middleware:**
- `authenticate` — verifies `Authorization: Bearer <jwt>` → `req.user = {userId, role}`
- `requireAdmin` — extends authenticate + checks `role === 'admin'`

### `db.js` (84 lines) — Database abstraction

**Dual backend:**
- `DATABASE_URL` set → PostgreSQL via `pg` (Railway)
- No `DATABASE_URL` → SQLite via `better-sqlite3` (local dev)

**Tables:**
```sql
users (id, discord_id UNIQUE, username, avatar_url, role DEFAULT 'user', created_at)
game_scores (id, user_id FK, game_code, score, total_rounds, played_at)
friendships (user_id FK, friend_id FK, status DEFAULT 'pending', created_at, PK(user_id, friend_id))
```

**Exports:** `generateId(), query(), get(sql, params), all(sql, params), run(sql, params)` — all async, works with both DB engines.

## Frontend — app/

### `page.tsx` (Home)
- Two CTAs: **Create Room** / **Join Room**
- Bottom nav: **Account** / **Leaderboard**

### `game/[code]/page.tsx` — Core game UI

**Sub-components:**

| Component | State | What it shows |
|-----------|-------|---------------|
| `WaitingRoom` | `waiting` | Player list, genre chips, rounds/time selectors (host only), Start button |
| `PlayingPhase` | `playing` | Timer, 48-bar visualizer (motion spring), progress bar (YouTube playback), guess input |
| `RoundResult` | `round_result` | Correct answer, artist, player scores, "Next round soon" |
| `GameFinished` | `finished` | Rankings podium, Play Again + Save Score buttons |
| `YouTubePlayer` | hidden | Hidden `<div>` with YouTube IFrame API, 0×0 size, auto-plays, loop on end |
| `Visualizer` | any | 48 `motion.div` bars, spring animation, gradient colors, heights randomized every 120ms |
| `ProgressBar` | any | `motion.div` width based on `getCurrentTime()/getDuration()`, polled every 200ms |

**Polling:** `fetchGameState(code)` every 1s. On `playing` → set `youtubeVideoId`. YouTube player polls `getCurrentTime()/getDuration()` every 200ms.

**Error handling:** 5 failed polls → show error with server URL.

### `login/page.tsx` — Discord OAuth
- If no token → "Login with Discord" button → redirects to `/api/auth/discord`
- Callback → token in URL → stored in `localStorage('blindtest_token')`
- If token exists → fetch `/api/users/me` → show profile card with avatar
- Links to Profile, Leaderboard, Admin (if admin), Logout

### `profile/page.tsx`
- Stats cards: Total Score, Best Game, Games Played, Friends
- Friend management: add by user ID, accept pending requests, remove friends
- Recent games list (last 10)

### `leaderboard/page.tsx`
- Sorted by total_score descending
- Medal icons for top 3
- Shows username, avatar, game count

### `admin/page.tsx`
- Table of all users with username, avatar, role
- Dropdown to change role (user/admin)
- Delete button (with confirmation)

### `create/page.tsx` + `join/page.tsx`
- Simple forms — create just asks for name (genres chosen in lobby)
- Join asks for room code + name

### `lib/api.ts` (103 lines) — API client

**Types:**
```typescript
GameState = waiting | playing | round_result | finished
RoomSettings = { rounds, roundTime, pauseTime }
Player = { id, name, score }
RoundResult = { round, correctAnswer, artist, albumImage }
Ranking = { rank, name, score }
User = { id, username, avatar_url, role, created_at }
```

**All API functions:** `fetchGenres()`, `createRoom()`, `joinRoom()`, `startGame()`, `submitAnswer()`, `fetchGameState()`, `updateSettings()`, `checkRoom()`, `searchYouTube()`, `getMe()`, `getMyScores()`, `getUserProfile()`, `getLeaderboard()`, `getFriends()`, `sendFriendRequest()`, `acceptFriendRequest()`, `removeFriend()`, `saveGameScore()`, `getAdminUsers()`, `updateUserRole()`, `deleteUser()`

**`fetchWithAuth(url, options?)`:** Automatically adds `Authorization: Bearer` from localStorage. On 401 → clears token.

## Improvement Ideas

### Audio
- Replace hidden YouTube IFrame with `youtubei.js` for direct audio URLs → enable Web Audio API real visualizer
- Spotify Web Playback SDK for higher quality (requires Premium)
- Support cover-art-only mode as alternative game variant

### Gameplay
- **Artist mode** — guess artist instead of song name
- **Album art rounds** — show album art, guess song/artist
- **Multi-genre per round** — mix genres in same game
- **Power-ups** — skip, 50/50, double points
- **Timed lobbies** — auto-start after countdown

### Social
- **Game invites** — share room code via Discord/WhatsApp
- **Friend games** — create room visible only to friends
- **Achievements** — streaks, perfect games, speed demon

### Technical
- **WebSockets** — replace polling for real-time game state
- **Rate limiting** — express-rate-limit for API endpoints
- **CSRF protection** — for auth endpoints
- **CORS restrict** — only allow frontend origin
- **Input sanitization** — against XSS in player names
- **Pagination** — for leaderboard with large user bases
- **Redis** — for room state instead of in-memory (for horizontal scaling)
- **E2E tests** — Playwright for game flow
- **CI/CD** — GitHub Actions for lint + test on PR
